Value Iteration

اضغط زر "📐 Value Iteration"
الحين نحل الـ MDP بشكل دقيق باستخدام معادلات بلمان من محاضرة 8.
Value Iteration يحسب V*(s) لكل الحالات — 6,250 حالة.
هذا model-based يعني يحتاج يعرف دالة الانتقال T(s,a,s').

الفرق عن Q-Learning: هذا يحسب كل شي مقدماً، بينما Q-Learning يتعلم بالتجربة.

A* Search

اضغط زر "🔍 A* Search"
الحين نستخدم A* Search من محاضرة 3 لإيجاد أفضل خطة تمارين.
A* يستخدم:
• g(n): التكلفة الفعلية من البداية
• h(n): تقدير المسافة للهدف (heuristic admissible)
• f(n) = g(n) + h(n): الأولوية
ونستخدم Graph Search بـ closed set عشان ما نزور حالات مكررة.
أشر على الخطة الناتجة
وجد أفضل خطة — مثلاً: يوم 1 جري + عجز سعرات، يوم 2 جري + بروتين... وهكذا حتى الهدف.

❓ أسئلة متوقعة من الدكتور وإجاباتها

ليه Q-Learning أفضل من Value Iteration هنا؟
لأن البيئة عندنا عشوائية (stochastic) — فيها ضوضاء ±20%. Q-Learning يتعلم من التجربة الفعلية ويتكيف مع العشوائية. بينما Value Iteration يستخدم انتقالات حتمية فقط.
ايش الفرق بين Q-Learning و Approximate Q-Learning؟
Q-Learning يخزن قيمة لكل (حالة، فعل) — جدول ضخم 1.4 مليون خانة. Approximate Q يستخدم 12 feature فقط مع أوزان — يعمم على حالات ما شافها.
ليه الـ heuristic بتاعت A* هي admissible؟
لأنها تحسب المسافة المتبقية للهدف بشكل مبسط — مثلاً: (الوزن الحالي - الوزن المطلوب) × 2. هذا أقل من التكلفة الحقيقية فما تبالغ أبداً.
ايش معنى ε-greedy؟
استراتيجية لحل مشكلة Exploration vs Exploitation من محاضرة 10. بنسبة ε نختار فعل عشوائي (استكشاف)، وبنسبة 1-ε نختار أفضل فعل (استغلال). ε يبدأ 1.0 وينزل تدريجياً لـ 0.05.
ايش هي معادلة بلمان؟
V*(s) = max_a [R(s,a,s') + γ·V*(s')] — يعني قيمة أي حالة = أفضل (مكافأة فورية + قيمة مخصومة للحالة الجاية). علاقة تكرارية نحلها بالتكرار حتى القيم تستقر.
ليه استخدمت discount factor γ = 0.95؟
عشان نعطي أهمية للمكافآت المستقبلية لكن نفضل القريبة. لو γ=0 يهتم بالحاضر فقط، لو γ=1 يهتم بالكل بالتساوي. 0.95 توازن جيد — المحاضرة 7 شرحت هذا.
كيف صممت دالة المكافأة؟
المكافأة تعتمد على هدف المستخدم: لنزول الوزن +3 لكل نقطة نزول وزن، لبناء العضل +3 لكل نقطة زيادة عضلات. عقوبات: -3 لو الإرهاق عالي (خطر إصابة)، -0.1 عقوبة زمنية لتشجيع الكفاءة.
