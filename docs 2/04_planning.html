1. الربط بالمنهج — Model-Based Planning (Lectures 3 & 8)

عكس الـ Q-Learning اللي يكتشف البيئة بالتجربة، وكلاء التخطيط (Planning Agents) يمتلكون خريطة كاملة للبيئة (Transition Model & Reward Function). من خلال هذه الخريطة، يقدرون يخططون للحل المثالي قبل اتخاذ أي خطوة فعلية.

في مشروعنا، בניنا שני Model-Based Agents:

  • ValueIterationAgent (Lecture 8): يحل معادلة Bellman بشكل دقيق (Exact Solution) لكل الـ States.
  • AStarSearchAgent (Lecture 3): يبحث عن الـ Optimal Path للهدف باستخدام Graph Search و Heuristic.

⚠️ Deterministic Assumption

خوارزميات التخطيط الكلاسيكية تفترض بيئة Deterministic (لا توجد عشوائية). عشان كدا، هذي הـ Agents تستخدم دالة env.get_deterministic_next_state() بدلاً من env.step() لتجنب الـ Stochastic Noise.

2. Value Iteration — value_iteration_agent.py

الـ Value Iteration (Lecture 8) هو خوارزمية Dynamic Programming تهدف لحساب הـ Optimal Value V*(s) لكل State في הـ MDP، ومن ثم استخراج الـ Optimal Policy π*(s).

2.1 State Space Reduction

الـ State Space الأصلي فيه 43,750 حالة. الـ Value Iteration يحتاج يمر على كل الـ States في كل حلقة (Iteration). عشان نسرّع الحسابات، شلنا הـ day_of_week (متغير دوري لا يؤثر على הـ Transitions الأساسية)، فصار الحجم:

Reduced State Space = 10 × 5 × 5 × 5 × 5 = 6,250 States

هذا يسرّع الخوارزمية 7 أضعاف، ويخليها تخلص في ~60 ثانية بدل 7 דقائق.

2.2 The Bellman Update Loop

الخوارزمية تطبق معادلة Bellman Optimality على كل הـ 6,250 States لحد ما يصير הـ Change (Δ) أصغر من theta = 0.01 (Convergence):

def solve(self):
    while True:
        delta = 0
        # 1. Loop over ALL 6,250 states
        for state in self.all_states:
            v_old = self.V[state]
            
            # 2. Find max Q-value across all 28 actions
            max_q = float('-inf')
            for action in range(self.num_actions):
                # Model-Based: We KNOW the exact next_state and reward!
                next_state, _ = self.env.get_deterministic_next_state(state, action)
                reward = self._compute_reward(state, next_state, action)
                
                # Q(s,a) = R(s,a,s') + γ * V(s')
                q_value = reward + self.gamma * self.V[next_state]
                max_q = max(max_q, q_value)
                
            # 3. V(s) = max_a Q(s,a)
            self.V[state] = max_q
            
            # 4. Update max change
            delta = max(delta, abs(v_old - max_q))
            
        # 5. Convergence check
        if delta < self.theta:
            break

2.3 Policy Extraction

بعد ما نلاقي הـ V*، نستخرج הـ Optimal Policy. הـ Agent لما ينسأل "شنو أسوي الحين؟"، ينفذ One-Step Lookahead:

def get_action(self, state):
    best_action = None
    best_value = float('-inf')
    
    # Calculate Q(s,a) for all actions and pick the max
    for action in self.env.valid_workout_indices:
        next_state, _ = self.env.get_deterministic_next_state(state, action)
        reward = self._compute_reward(state, next_state, action)
        
        value = reward + self.gamma * self.V[next_state]
        if value > best_value:
            best_value = value
            best_action = action
            
    return best_action

3. A* Search — search_agent.py

من Lecture 3: הـ A* Search هو Informed Search Algorithm يضمن إيجاد הـ Optimal Path אם كانت הـ Heuristic Admissible.

3.1 SearchNode Design

كل Node في شجرة البحث يحتوي على:

  • state: حالة הـ Environment.
  • actions: المسار (List of actions) من البداية لهذه הـ Node.
  • g_cost: التكلفة الفعلية حتى الآن (نستخدم הـ Negative Reward كـ Cost لأن A* يبحث عن הـ Minimum Cost).
  • h_cost: تقدير المسافة للهدف (الـ Heuristic).
  • f_cost = g_cost + h_cost: הـ Priority اللي نستخدمها في הـ Priority Queue.
  • accumulators: الـ Accumulator Pattern State (مهم جداً).

✅ The Accumulator Propagation

לما A* يولد Successor Nodes، لازم ينسخ ويمرر הـ Accumulators للـ Child Node. بدون تمرير הـ Accumulators الفراكشنية، A* بيحسب weight + (-0.05) = weight، وراح يعتقد إن הـ Diet ماله أي فائدة، ويفشل في إيجاد مسار للهدف.

3.2 הـ Graph Search Algorithm

طبقنا الـ Graph Search (من Lecture 3) باستخدام הـ closed Set لمنع الـ Agent من زيارة نفس הـ State مرتين والوقوع في Infinite Loops:

def search(self, start_state):
    # 1. Initialize Frontier (Priority Queue) with start node
    frontier = [start_node]
    heapq.heapify(frontier)
    
    # 2. Initialize Closed Set (Graph Search optimization)
    closed = set()
    
    while frontier and nodes_expanded < self.beam_width:
        # 3. Pop node with lowest f(n)
        current = heapq.heappop(frontier)
        
        if self._is_goal(current.state):
            return current.actions  # Solution Found!
            
        # 4. Skip if already visited
        if current.state in closed:
            continue
        closed.add(current.state)
        
        # 5. Expand successors
        for action in range(NUM_ACTIONS):
            next_state, new_acc = self.env.get_deterministic_next_state(
                current.state, action, current.accumulators
            )
            
            # g(n') = g(n) + step_cost (Negative Reward)
            step_reward = self._estimate_reward(current.state, next_state, action)
            new_g = current.g_cost - step_reward
            
            # h(n')
            new_h = self.env.state_distance_to_goal(next_state)
            
            successor = SearchNode(next_state, new_g, new_h, new_acc)
            heapq.heappush(frontier, successor)

3.3 Heuristic Admissibility

كما شُرح في ملف الـ Environment (العضو 2)، دالة state_distance_to_goal() تعتبر Admissible Heuristic لأنها دائماً تقلل (Underestimate) من التكلفة الحقيقية (تفترض الوصول للهدف بأقصى سرعة ممكنة وبدون أي Fatigue Penalties).

4. مقارنة: A* Search vs Value Iteration

الخاصيةA* SearchValue Iteration
الناتج (Output) مسار واحد (Sequence of Actions) لـ State واحدة. Policy كاملة (أفضل Action) لكل הـ 6,250 States.
نطاق البحث يبحث محلياً (Local Search) باتجاه الهدف. يحسب الـ Value لكل State في النظام (Global).
سرعة التنفيذ سريع جداً (أقل من 3 ثوانٍ). بطيء (يأخذ ~60 ثانية لحل الـ State Space الكامل).
التعامل مع הـ Stochasticity يفشل إذا كانت البيئة Stochastic (يحتاج Deterministic Path). نظرياً يقدر يتعامل مع Stochasticity لو הـ Model كان يعطي Probabilities، لكن هنا استخدمنا Deterministic للتبسيط وتجنب Loop الـ Probabilities المعقد.
