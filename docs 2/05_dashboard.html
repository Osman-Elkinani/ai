واجهة المستخدم — Presentation Layer

1. معمارية الـ Presentation Layer

هذه الطبقة هي واجهة المشروع. دورها هو ربط المستخدم بـ الـ AI Agents اللي شغالة في الـ Backend وعرض النتائج (الـ Metrics والـ Q-Values) بطريقة مرئية سهلة الفهم.

مكونات هذه الطبقة:

  • Backend (Flask API): app.py — يستقبل الطلبات (HTTP Requests)، يشغل הـ Agents، ويرجع النتائج كـ JSON.
  • Frontend Logic: dashboard.js — يتحكم في التفاعل، يرسل הـ Requests للـ API، ويرسم הـ Charts.
  • UI/UX: dashboard.html و style.css — هيكلة الصفحة والتصميم المرئي.

2. Backend API — app.py

בنينا 6 API Endpoints لتسهيل التواصل بين الـ Frontend والـ AI:

EndpointMethodالوظيفة
/ GET يعرض صفحة dashboard.html الرئيسية.
/api/profile POST يغير הـ User Profile (مثلاً من Beginner לـ Advanced). يعمل Reset לـ Environment.
/api/train POST يستقبل عدد الـ Episodes، يشغل הـ RL Agent، ويرجع הـ Metrics כـ JSON.
/api/solve_exact POST يشغل Value Iteration أو A* Search ويرجع الحل الكامل.
/api/recommendation GET يطلب من הـ Agent المختار تحديد הـ Best Action للـ State الحالية.
/api/take_action POST يأخذ הـ Action اللي اختاره المستخدم وينفذ env.step(action).

💡 Stateless API vs Stateful Backend

سيرفر Flask عادة يكون Stateless (لا يحفظ الحالة بين الـ Requests). لكن لأن التدريب مستمر (Continuous Training)، احتجنا نحفظ הـ env والـ agents כـ Global Variables في app.py عشان ما تضيع הـ Q-Table بعد كل Request.

3. Frontend Logic — dashboard.js

ملف הـ JS فيه أكثر من 700 سطر כود. أهم ميزاته الهندسية:

3.1 Chunked Training (Batching)

المشكلة: لو طلبنا من Flask يتدرب 5000 Episode دفعة واحدة، הـ Browser سيعلق (Freeze) لمدة 15 ثانية ينتظر الـ Response، والمستخدم بيظن إن الموقع معلّق.

الحل (Chunked Training): قسمنا التدريب إلى دفعات (Batches). كل دفعة 50 Episode. الـ JS يرسل Request لتدريب 50، يستقبل النتائج، يحدث הـ Chart، ثم يرسل Request للـ 50 اللي بعدها.

async function trainBatch() {
    // 1. Check if training was stopped
    if (!isTraining) return;

    // 2. Train next 50 episodes
    const response = await fetch('/api/train', { episodes: 50 });
    const data = await response.json();

    // 3. Update Charts
    updateCharts(data.metrics);

    // 4. Update Progress Bar
    const progress = (currentEpisode / totalEpisodes) * 100;
    updateProgressBar(progress);

    // 5. Schedule next batch (Recursion)
    if (currentEpisode < totalEpisodes) {
        setTimeout(trainBatch, 50); // Small delay to unblock UI thread
    }
}

3.2 زر التوقف الإيقاف (Stop Training)

لأننا نستخدم Chunked Training، صار عندنا القدرة نضيف زر "إيقاف التدريب".

let isTraining = false;

// When Stop button is clicked:
function stopTraining() {
    isTraining = false;  // This breaks the Recursion in trainBatch()
    enableButtons();
    showNotification("Training Stopped by User");
}

الزر يغير הـ Flag isTraining = false، ولما الـ Batch الحالي يخلص، دالة trainBatch بتشيك الـ Flag وتوقف الـ Recursion.

3.3 Live Charts (Chart.js)

استخدمنا مكتبة Chart.js لرسم الـ Learning Curves في الـ Real-Time. الـ Charts تتحدث ديناميكياً مع كل Batch يرجع من הـ Server، مما يعطي إحساس للمستخدم أن הـ AI "يتعلم أمامه".

4. UI/UX & Theming — Professional Blue

التصميم مبني بالكامل على CSS Vanilla بدون أي Framework (مثل Tailwind أو Bootstrap) لإثبات القدرة على التحكم الكامل بالـ Styles.

4.1 CSS Variables (Design Tokens)

استخدمنا הـ Variables לضمان Consistency في الألوان في كل الصفحة ولتسهيل تغيير הـ Theme مستقبلاً:

:root {
    --bg-color: #0a0f1e;        /* Deep Navy Blue Background */
    --surface-1: #141c30;       /* Slightly lighter for cards */
    --accent-blue: #3b82f6;     /* Primary Action Color */
    --text-primary: #eef2f9;    /* Clean White Text */
    --neon-glow: rgba(59, 130, 246, 0.4);
}

4.2 Glassmorphism & Modern Aesthetics

اعتمدنا على مظهر הـ Glassmorphism (تأثير الزجاج) للبطاقات (Cards) باستخدام الـ backdrop-filter וالـ Borders الشفافة:

.dashboard-card {
    background: var(--surface-1);
    border: 1px solid rgba(255, 255, 255, 0.05);
    border-radius: 16px;
    box-shadow: 0 8px 32px rgba(0, 0, 0, 0.2);
    /* Micro-animation on hover */
    transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.dashboard-card:hover {
    transform: translateY(-2px);
    box-shadow: 0 12px 40px var(--neon-glow);
}

5. Real-Time Metrics Display

الـ Dashboard يعرض 3 أنواع من הـ Metrics المستمرة:

  1. State Metrics: (Fitness, Energy, Weight, Muscle, Fatigue). يتم جلبها عبر /api/recommendation وعرضها כـ Progress Bars دائرية وأفقية.
  2. Q-Values: نعرض أفضل 3 Actions وقيمة الـ Q-Value (الـ Expected Reward) لكل واحد منهم. هذا يخلي הـ AI "شفاف" (Explainable AI) וالمستخدم يشوف ليش הـ Agent اختار هذا التمرين.
  3. Agent Performance: הـ Learning Curve (الـ Reward יرتفع) וالـ State Coverage (كم State استكشفها).
